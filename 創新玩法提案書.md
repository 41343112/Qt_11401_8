## 🎯 專案概述

### 背景說明

Qt Chess 是一款功能完整的國際象棋遊戲，支援雙人對戰、AI 對弈和線上對戰。為了增加遊戲的趣味性和挑戰性，我們提出引入「地形系統」的創新玩法，靈感來自經典的踩地雷遊戲。

### 核心目標

- **創新性**：結合傳統國際象棋與踩地雷機制，創造獨特的遊戲體驗
- **可選性**：玩家可自由選擇啟用或關閉任何玩法組合
- **平衡性**：不改變棋子的基本走法，保持國際象棋的核心規則
- **趣味性**：增加戰略深度和不確定性，提升遊戲的重複遊玩價值

### 核心約束

**重要原則：不能改變棋子的走法**

所有創新玩法僅影響棋盤環境和額外效果，不修改任何棋子的移動規則。國王、皇后、城堡、主教、騎士、兵的移動方式完全保持原始國際象棋規則。

## 💡 設計理念

### 踩地雷遊戲機制

借鑒經典踩地雷遊戲的核心概念：

1. **隱藏資訊**：棋盤上的地形效果初始為隱藏狀態
2. **揭露機制**：當棋子移動到某格時，該格的地形效果被揭露
3. **數字提示**：安全區域會顯示周圍危險格子的數量
4. **連鎖反應**：移動到安全區域可能觸發連鎖揭露
5. **風險與回報**：某些地形提供獎勵，某些造成懲罰

### 與國際象棋的結合

- 地形系統作為額外的戰略層
- 玩家需要在國際象棋戰術和地形探索間取得平衡
- 增加遊戲的不確定性和刺激感
- 提供更多戰術選擇和策略深度

## 🎮 六大創新玩法

### 1. 經典地雷模式 (Classic Mines)

**模式代號：** GameVariant::ClassicMines

#### 玩法說明

棋盤上隨機分布地雷格子（類似踩地雷遊戲）。當棋子移動到地雷格時，該棋子被立即消滅。

#### 遊戲機制

- **地雷數量**：10-15 個地雷隨機分布
- **揭露方式**：棋子移動到某格時揭露該格
- **安全區域**：非地雷格子顯示周圍 8 格的地雷數量（0-8）
- **視覺效果**：
  - 未揭露格子：正常顯示
  - 地雷格子：💣 符號（踩到後顯示）
  - 安全格子：顯示數字（0-8）

#### 策略要素

- 玩家需要根據已揭露的數字推測地雷位置
- 可以利用對手的移動來探索地圖
- 王、后等重要棋子需要謹慎移動
- 可以用兵作為「探雷兵」

#### 遊戲影響

- **風險管理**：高價值棋子移動需更謹慎
- **資訊戰**：利用對手的移動獲取地形資訊
- **戰術調整**：可能需要犧牲低價值棋子探路

---

### 2. 安全區域模式 (Safe Zones)

**模式代號：** GameVariant::SafeZones

#### 玩法說明

棋盤劃分為安全區域和危險區域。安全區域顯示周圍危險格子的數量提示，類似踩地雷的數字系統。

#### 遊戲機制

- **區域劃分**：
  - 安全區域：60% 的格子
  - 危險區域：40% 的格子（包含各種地形）
- **數字提示系統**：
  - 0：周圍 8 格都安全
  - 1-8：周圍有相應數量的危險格子
- **自動揭露**：踩到 0 的格子時，自動揭露周圍安全區域

#### 策略要素

- 根據數字提示規劃移動路線
- 利用自動揭露機制快速探索安全路徑
- 創造對手必須進入危險區域的局面

#### 遊戲影響

- **規劃能力**：需要根據數字進行邏輯推理
- **空間控制**：安全區域成為戰略要地
- **壓迫戰術**：迫使對手進入未知區域

---

### 3. 獎勵移動模式 (Bonus Moves)

**模式代號：** GameVariant::BonusMoves

#### 玩法說明

棋盤上隨機分布獎勵格子（⭐）。當棋子移動到獎勵格時，該玩家可以立即再移動一次（額外回合）。

#### 遊戲機制

- **獎勵格數量**：6-8 個隨機分布
- **觸發條件**：棋子移動到獎勵格（揭露時觸發）
- **額外回合**：
  - 可以移動任意己方棋子
  - 同一回合內最多獲得 1 次額外移動
  - 額外移動時踩到獎勵格不再觸發
- **視覺效果**：⭐ 星星符號

#### 策略要素

- 爭奪獎勵格的控制權
- 利用額外移動建立連續進攻
- 計算對手可能獲得的額外移動

#### 遊戲影響

- **節奏控制**：獎勵移動可以打亂正常節奏
- **連續攻擊**：創造連續將軍機會
- **地圖控制**：獎勵格成為爭奪焦點

---

### 4. 護盾能力模式 (Shield Power)

**模式代號：** GameVariant::ShieldPower

#### 玩法說明

棋盤上分布護盾格子（🛡️）和地雷格子。獲得護盾的棋子可以免疫一次地雷傷害。

#### 遊戲機制

- **地形分布**：
  - 護盾格：5-6 個
  - 地雷格：10-12 個
- **護盾效果**：
  - 踩到護盾格時獲得護盾狀態
  - 有護盾的棋子踩到地雷時，消耗護盾而不受傷
  - 每個棋子最多持有 1 個護盾
- **視覺標記**：有護盾的棋子顯示特殊標記（🛡️圖示）

#### 策略要素

- 先獲取護盾再探索危險區域
- 保護重要棋子不被地雷消滅
- 計算護盾的戰略價值

#### 遊戲影響

- **風險降低**：降低探索的風險
- **資源管理**：護盾是珍貴的一次性資源
- **戰術規劃**：規劃護盾的使用時機

---

### 5. 傳送混亂模式 (Teleport Chaos)

**模式代號：** GameVariant::TeleportChaos

#### 玩法說明

棋盤上分布傳送格子（🌀）。當棋子移動到傳送格時，會被隨機傳送到棋盤上的另一個空格。

#### 遊戲機制

- **傳送格數量**：8-10 個
- **傳送規則**：
  - 隨機傳送到任意空白格子
  - 不會傳送到有棋子的格子
  - 不會傳送到起始位置
  - 傳送後立即揭露目標位置的地形
- **連鎖傳送**：如果傳送到的位置也是傳送格，立即再次傳送

#### 策略要素

- 利用傳送格快速調動棋子
- 計算傳送的風險與回報
- 可能將對手的棋子困在不利位置

#### 遊戲影響

- **位置不確定**：增加戰局的不可預測性
- **快速機動**：提供快速移動的可能
- **混亂要素**：打破常規的開局和中局

---

### 6. 漸進地雷模式 (Progressive Mines)

**模式代號：** GameVariant::ProgressiveMines

#### 玩法說明

遊戲開始時地雷較少，隨著遊戲進行，棋盤上會動態生成新的地雷，增加遊戲難度。

#### 遊戲機制

- **初始地雷**：5 個
- **生成規則**：
  - 每 5 個回合（10 步棋）新增 2 個地雷
  - 地雷只生成在未揭露的空白格子
  - 不會生成在有棋子的格子
  - 最多 20 個地雷
- **難度曲線**：遊戲後期地雷密度顯著增加

#### 策略要素

- 早期積極探索和展開
- 後期需要更謹慎的移動
- 時間壓力：避免拖入後期

#### 遊戲影響

- **時間壓力**：鼓勵快速決戰
- **動態環境**：局勢隨時間改變
- **適應能力**：需要根據地雷密度調整戰術

```cpp
// 地形類型枚舉
enum class TerrainType {
    None = 0,        // 無地形
    Mine = 1,        // 地雷
    Safe = 2,        // 安全區
    Bonus = 3,       // 獎勵格
    Shield = 4,      // 護盾格
    Teleport = 5,    // 傳送格
    Revealed = 6     // 已揭露
};

// 遊戲變體枚舉
enum class GameVariant {
    ClassicMines = 0,
    SafeZones = 1,
    BonusMoves = 2,
    ShieldPower = 3,
    TeleportChaos = 4,
    ProgressiveMines = 5
};

// 新增成員變數
private:
    bool m_gameVariants[6];                          // 6種變體啟用狀態
    std::vector<std::vector<TerrainType>> m_terrain; // 地形數據
    std::vector<std::vector<bool>> m_terrainRevealed; // 揭露狀態
    std::vector<std::vector<int>> m_adjacentMines;    // 周圍地雷數
    std::vector<std::vector<bool>> m_pieceShields;    // 護盾狀態
    bool m_whiteExtraMove;                           // 白方額外移動
    bool m_blackExtraMove;                           // 黑方額外移動
```

### 遊戲流程整合

#### 移動棋子時的處理流程

```
1. 玩家選擇移動（from -> to）
2. 驗證移動合法性（原有邏輯）
3. 執行移動
4. 檢查目標格子地形
   ├─ 如果未揭露：揭露地形
   ├─ 根據地形類型執行效果：
   │   ├─ Mine：檢查護盾 → 消滅棋子或消耗護盾
   │   ├─ Safe：顯示周圍地雷數 → 如果為0則連鎖揭露
   │   ├─ Bonus：給予額外移動機會
   │   ├─ Shield：給予護盾
   │   └─ Teleport：隨機傳送
5. 更新遊戲狀態
6. 檢查遊戲結束條件
7. 如果是漸進地雷模式：檢查是否需要添加新地雷
```

#### 視覺效果實作

```cpp
void Qt_Chess::displayTerrainOnSquare(QPushButton* square, 
                                       int row, int col) {
    if (!m_chessBoard.isTerrainRevealed(row, col)) {
        return; // 未揭露，不顯示
    }
    
    TerrainType terrain = m_chessBoard.getTerrain(row, col);
    QString terrainSymbol;
    QString terrainColor;
    
    switch (terrain) {
        case TerrainType::Mine:
            terrainSymbol = "💣";
            terrainColor = "#FF5252";
            break;
        case TerrainType::Bonus:
            terrainSymbol = "⭐";
            terrainColor = "#FFD700";
            break;
        case TerrainType::Shield:
            terrainSymbol = "🛡️";
            terrainColor = "#2196F3";
            break;
        case TerrainType::Teleport:
            terrainSymbol = "🌀";
            terrainColor = "#9C27B0";
            break;
        case TerrainType::Safe:
            int mines = m_chessBoard.countAdjacentMines(row, col);
            terrainSymbol = QString::number(mines);
            // 根據地雷數量設定顏色
            terrainColor = getMineCountColor(mines);
            break;
    }
    
    // 在格子角落顯示地形標記
    square->setStyleSheet(square->styleSheet() + 
        QString("; qproperty-icon: url(%1);").arg(terrainSymbol));
}
```

## ⚖️ 遊戲平衡性分析

### 變體組合建議

#### 初學者友善組合

- ✅ 安全區域模式
- ✅ 護盾能力模式
- ❌ 其他模式

**特點**：提供數字提示和保護機制，降低風險

#### 標準遊戲組合

- ✅ 經典地雷模式
- ✅ 安全區域模式
- ✅ 護盾能力模式
- ❌ 其他模式

**特點**：平衡的風險與回報，類似標準踩地雷

#### 高難度組合

- ✅ 經典地雷模式
- ✅ 傳送混亂模式
- ✅ 漸進地雷模式
- ❌ 其他模式

**特點**：高度不確定性，需要快速決戰

#### 策略深度組合

- ✅ 安全區域模式
- ✅ 獎勵移動模式
- ✅ 護盾能力模式
- ❌ 其他模式

**特點**：強調資源管理和戰術規劃

### 禁止組合

某些變體組合可能造成遊戲過於混亂或不公平：

#### 不建議同時啟用

1. **傳送混亂 + 漸進地雷**：過度隨機，策略性降低
2. **所有 6 種同時啟用**：規則過於複雜，影響遊戲體驗

### 平衡性調整參數

#### 可調整參數

| 參數 | 預設值 | 調整範圍 | 說明 |
|-----|--------|----------|------|
| 地雷數量 | 12 | 5-20 | 影響探索風險 |
| 獎勵格數量 | 6 | 4-10 | 影響額外移動機會 |
| 護盾格數量 | 5 | 3-8 | 影響保護能力 |
| 傳送格數量 | 8 | 5-12 | 影響位置不確定性 |
| 漸進地雷增長速度 | 每5回合+2 | 每3-10回合+1-3 | 影響時間壓力 |

## 📅 實作時程規劃

## 📝 附錄

### A. 技術名詞解釋

| 術語 | 解釋 |
|-----|------|
| 地形系統 | 棋盤格子附加的特殊屬性，影響遊戲進行 |
| 揭露機制 | 類似踩地雷，移動到格子時顯示該格的地形類型 |
| 連鎖揭露 | 踩到安全區（0地雷）時，自動揭露周圍的安全區域 |
| 遊戲變體 | 可選的遊戲規則變化，改變遊戲體驗但不改變基本規則 |

## ✅ 結論

本提案設計了 6 種創新的地形系統遊戲變體，靈感來自經典的踩地雷遊戲。這些變體保持了國際象棋的核心規則（不改變棋子走法），同時增加了戰略深度、不確定性和趣味性。

### 核心優勢

1. **創新性高**：結合兩種經典遊戲機制
2. **可選性強**：玩家完全自主選擇
3. **平衡性好**：不破壞國際象棋的核心玩法
4. **擴展性佳**：可持續添加新變體

### 實作可行性

- 技術架構清晰，易於實作
- UI 設計簡潔，整合順暢
- 時程規劃合理，6.5週可完成
- 風險可控，應對策略明確

### 預期影響

透過這些創新玩法，Qt Chess 將提供更豐富的遊戲體驗，吸引更廣泛的玩家群體，並在市場上建立差異化優勢。
